Brand Style Guide

ORLA — Identity, color, typography, posture

01 · Identity

Brand Identity

ORLA
Pâtisserie Algérienne — Tools, Ingrédients, Packaging

ORLA is an Algerian ecommerce shop specializing in pastry and sweets making tools, cooking ingredients, and boxes/packaging. The name evokes warmth, craft, and the golden hues of traditional Algerian pastry. The brand speaks to home bakers and professionals who value quality tools and authentic ingredients.

04 · Posture

Posture Rules

Six rules that define how ORLA looks and feels. These are non-negotiable.

  1. 1
    Warm over clinical
    The Cream/Lemon palette reads warm. Never override to cool gray or pure white backgrounds. The brand should feel like a pastry shop — inviting, golden, handcrafted — not a hospital or fintech app.
  2. 2
    Accent restraint
    Crimson Violet (—accent) appears at most twice per visible screen: one primary CTA + one supplementary element (badge, link, or highlight). Overuse kills its impact.
  3. 3
    Heading discipline
    All h1–h3 use Orla Serif. h4–h6 use Montserrat 600. Never mix serif into UI chrome (sidebar labels, button text, table headers).
  4. 4
    RTL-first spacing
    All directional padding/margin uses logical properties (padding-inline, margin-block). No hardcoded left/right spacing. Arabic is a first-class language.
  5. 5
    Dark mode is not inverted
    Dark mode uses deep olive backgrounds, not black. The warmth is preserved. Never use pure black (#000) or pure white (#fff) in dark mode surfaces.
  6. 6
    Data density in dashboard, breathing room in storefront
    Admin uses tighter spacing (8–12px gaps) for data density. Storefront uses generous spacing (16–24px gaps) for browsing comfort.

05 · Motion

Motion Principles

Motion should feel purposeful and warm. Three principles guide all animation decisions.

Purposeful
Every animation must have a reason: guide attention, show relationships, or provide feedback. Decorative motion is noise.
Brief
Most transitions are 150–200ms. Only page transitions reach 300ms. Nothing should feel slow or ponderous.
Respectful
prefers-reduced-motion: reduce kills all animations. Accessibility is not optional. The interface must work without motion.

07 · Accessibility

RTL Considerations

Arabic is a first-class language. All spacing and layout must work bidirectionally.

RuleImplementation
Logical propertiesAll directional padding/margin uses padding-inline, margin-block. No hardcoded left/right.
Font fallbackArabic text uses Montserrat body stack; fallback to system Arabic fonts (Noto Sans Arabic, etc.)
dir attributedir="rtl" applied at <html> or container level
Radix direction@radix-ui/react-direction handles component-level RTL
Spacing mirroringAll spacing/border-direction utilities mirror automatically via Tailwind RTL support

08 · Layout

Elevation Hierarchy

Five-level hierarchy from flat content to toast notifications. Each level uses increasing shadow depth.

LevelShadowUse
1. Flat--shadow-xsBackground content, inline text
2. Raised--shadow-smCards, panels
3. Overlay--shadow-mdDropdowns, popovers
4. Modal--shadow-lgDialogs, sheets
5. Toast--shadow-xlNotifications

Overlay Pattern

All overlay components (dialog, sheet, dropdown, popover) share a backdrop: bg-black/80 with backdrop-blur-xs. Content panels use glassmorphism: semi-transparent background with blur layer and subtle ring border. Dialogs use rounded-4xl; dropdowns/popovers use rounded-2xl.

09 · Accessibility

Focus States

Keyboard-first focus visibility. Mouse users don't see rings; keyboard users do.

PatternCSSUse
Standard focusfocus-visible:ring-[3px] focus-visible:ring-ring/50All interactive elements
Error focusaria-invalid:border-destructiveForm validation states
Focus withingroup-data-[disabled=true]:pointer-events-noneCompound components
Do
Use focus-visible (not focus) so mouse clicks don't show rings. 3px ring with 50% opacity works on both light and dark backgrounds.
Don't
Never remove focus outlines entirely. Keyboard users must be able to navigate your interface.
